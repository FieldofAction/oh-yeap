# CANON.md — Field of Action
> The compounding substrate. **Read before generating. Write back after deciding.**
> This file is the asset. Outputs are downstream of it. If a decision isn't captured here,
> it didn't compound — it just happened.

---

## 0 · Operating discipline (plan / work)

- **80 / 20.** Most of the effort lives in this file (plan, constrain, review). Execution is the cheap 20%.
- **Read first.** Before producing anything, load §1–§4 as constraints. Generation that ignores the canon is noise, not work.
- **Write back.** After any non-trivial decision, append to §5. After anything that didn't fit, log it in §6.
- **One verb at a time.** Each agent (§3) does its one job. Don't let Action evaluate or Grace produce.

## 1 · ASU — invariants (what must remain true)

These do not change to suit a brief. A brief that requires breaking one is the wrong brief.

- **Coherence** — every element traces to the same underlying intent.
- **Intent integrity** — the original signal survives every handoff.
- **Non-arbitrary** — every decision has a reason traceable to this file.
- **Constraint alignment** — constraints come from the system, never from convenience or habit.

**Evolution rule:** this section changes *only* through repeated evidence + a validated anomaly (see §6).
No single instance overrides governance. Slow and deliberate, or it isn't canon.

## 2 · Pre-load constraints (active, project-agnostic)

> These are the constraints the agent loads *before* exploring options.

**Register**
- Unburdened: authorial presence without sacrificing craft gravity.
- Anti-chameleon: write/design from the center outward, never reverse-engineered from the brief or the audience.

**Type** — the five cuts of the FOA Writing System
- Display: Playfair Display
- Utility / data / labels (Mono): IBM Plex Mono
- Text / interface (Sans): IBM Plex Sans
- Slab / Bridge: unheld by design — the system runs on four held cuts; this slot stays the negative space until a brief forces it.
- Glyphs: FOA custom glyph set (Glyph Compositor output).

**Color**
- Ink: `#0C0D10`
- Paper: `#F2EBDF`
- Accent: `#2F5BFF` (cobalt, tuned slightly deeper) — carries exactly one meaning: it marks the single live / active / chosen element on a surface (the node that activates, the live indicator). Everything else holds at ink-on-paper.

**Surface rules (Relational Design Canon)**
- Clarity over density · guided interaction · every action has visible feedback · design for the moment the user is in · emotional coherence.

## 4 · Failure-mode gate (Grace checks output against these before it lands)

> The design equivalent of a test suite. If an output trips one of these, it returns to WIP — it does not reach Cache.
> Full definitions, tells, and corrections live in the Failure Modes Canon. The lines below are the gate form.

**Inflation modes (4) — what reality omits from the model (external)**
1. **Power Asymmetry** (I-01) — conditions are designed but no one is entering; the missing thing is felt permission, not formal access.
2. **Hidden Incentives** (I-02) — coherent conditions, incoherent behavior; participants are optimizing for something the relational frame can't see.
3. **Scale Effects** (I-03) — worked at 6, broke at 60; the field thinned past the scale the designer's attention can hold.
4. **Adversarial Conditions** (I-04) — better conditions, consistently undermined; an actor is working against the field, not merely misaligned.

**Shadow modes (7) — where the framework exceeds itself (internal; each is the cost of a real value)**
1. **Infinite Deferral** (S-01) — preparation that cannot end; the not-yet-ready is protecting against release. *(shadow of Trust Emergence)*
2. **Hidden Control** (S-02) — openness that never surprises its author; the container is steering the outcome. *(shadow of Design Conditions)*
3. **Over-Attunement** (S-03) — everyone feels good, nothing is decided; harmony bought with avoided truth. *(shadow of Attunement)*
4. **Moral Aestheticization** (S-04) — the process is more interesting than the output; elegance preferred over effect. *(shadow of Integrity of Practice)*
5. **Anti-Structure Bias** (S-05) — open but not operational; form refused where form was needed. *(shadow of Resist Imposition)*
6. **Fragility to Speed** (S-06) — patience curdling into passivity while a decision is waited on. *(shadow of Patience)*
7. **Reality Fragility** (S-07) — every intervention absorbed without effect; relational means applied to a context that won't hold them. *(shadow of Ideal Conditions)*

**Compounding-specific watch:** *self-imitation* — output resembles past Cache entries more than present Field signal. This is the failure unique to a compound system. Flag it as an anomaly (§6), not a pass.

### 2026-06-28 · Loop Period property
- Context / signal: Sunday digest flagged "drag" on open loops by reading every loop against the weekly clock. Daniel corrected: loops run on different periods, and judging long-period work at a weekly clip mislabels healthy work as a stall.
- Decision: add a Period property to every loop in LOOPS.md. Three values — Immediate (closes within a cycle), Medium (advances by stages over several cycles), Long-line (runs months, often gated on a decision or an emergence). The digest judges progress against a loop's own period, not the cadence.
- Rejected: uniform closure-pressure across all loops (fails Intent integrity — manufactures false drag and pushes premature closure on long-line work such as The Green Sun).
- Principle it sets: progress is period-relative. Surfaces to LOOPS.md as a required Period field. Grace reads drag against period, not calendar.
- Agent(s): Field (read the anomaly) → Open (re-sort loops) → Freedom Embassy (set the property).

## 6 · Anomaly register (Anomaly Inquiry Protocol)

### 2026-06-28 · Location carried as a thread for weeks without closing

2. Improving or degrading: neither — it was miscategorized, which read as degrading (false drag).
3. Deeper truth it may reveal: it was never a thread that advances; it is a gate that closes on a decision.
4. Scope: regional (the loop registry) → systemic (loop typing).
5. Disposition: INTEGRATE.
- Changed: LOOPS.md gains a GATE loop type; Location reclassified as a long-line gate with wake triggers. Evidence trail: weekly digests Jun 8–28, Anomaly Inquiry pass run 2026-06-28.

## 7 · Signal-fit test (Grace's pass/fail, since design has no unit tests)

Answer before anything reaches Cache. A "no" sends it back to WIP.

- Does it preserve the signal that was sensed in Field? (Signal integrity)
- Could it be mistaken for anyone else's work? If yes — it's templated, not authored.
- Does it survive reduction? (identity holds at 16px / in plain text)
- Is every element doing one job? Remove one accessory — is it stronger?
- Is this present-signal, or am I repeating a past Cache entry? (self-imitation check)
